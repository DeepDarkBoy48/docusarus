# Git 教程

欢迎来到 Git 教程！在这里，你将学习到 Git 的核心概念和实践技巧。

## 什么是 Git？

Git 是一个分布式版本控制系统，它可以跟踪文件的变化，并允许多人协作开发项目。它由 Linus Torvalds 于 2005 年创建，现在已经成为最流行的版本控制系统之一。

## 为什么使用 Git？

- **版本控制**：跟踪和记录文件的修改历史
- **协作开发**：支持多人同时开发项目
- **分支管理**：允许并行开发不同特性
- **代码备份**：确保代码安全性

## 基础概念

### 仓库（Repository）

仓库是 Git 用来存储项目的地方，包含了项目的所有文件和修改历史。

### 提交（Commit）

提交是对文件修改的一次记录，每个提交都有：

- 唯一的标识符
- 提交信息
- 时间戳
- 作者信息

### 分支（Branch）

分支允许你从主开发线上分离出来，独立开发特性。

## 常用命令

### 初始化仓库

```bash
git init
```

### 克隆仓库

```bash
git clone <repository-url>
```

### 添加文件到暂存区

```bash
git add <file-name>
# 或添加所有文件
git add .
```

### 提交更改

```bash
git commit -m "提交信息"
```

### 查看状态

```bash
git status
```

### 查看提交历史

```bash
git log
```

## 分支操作

### 创建分支

```bash
git branch <branch-name>
```

### 切换分支

```bash
git checkout <branch-name>
# 或使用新命令
git switch <branch-name>
```

### 创建并切换分支

```bash
git checkout -b <branch-name>
# 或使用新命令
git switch -c <branch-name>
```

## 远程操作

### 添加远程仓库

```bash
git remote add origin <repository-url>
```

### 推送到远程仓库

```bash
git push origin <branch-name>
```

### 从远程仓库拉取

```bash
git pull origin <branch-name>
```

## 最佳实践

1. **经常提交**：保持小而频繁的提交
2. **写好提交信息**：清晰描述修改内容
3. **使用分支**：为不同特性创建独立分支
4. **定期同步**：经常与远程仓库同步
